THE SOLUTION

Phase-Based Orchestration

Clear gates, incremental validation, human oversight

1. Plan Project ‚Üí Break into 3-7 phases with completion criteria
2. For each phase:
    a. Design (zen-architect)
    b. Validate Design (antagonist) ‚Üí [HUMAN APPROVES]
    c. Implement (modular-builder, max 800 turns)
    d. Validate Implementation (antagonist) ‚Üí [HUMAN APPROVES]
    e. Mark complete, proceed to next phase
3. Complete ‚Üí Production-ready system with confidence

LESSONS LEARNED

What Works vs What Fails

‚úÖ Success Patterns (Lifeline)

  • Clear phases with completion criteria
  • Focused sessions (never exceeded 850 turns)
  • Simple requirements from user perspective
  • Incremental validation at phase boundaries
  • Proactive documentation before building
  • Human approval at critical gates

‚ùå Failure Patterns (Word)

  • Task database without clear progression
  • Session bloat (sessions hit 1500+ turns)
  • Complex design specs upfront
  • Terminal validation (too late to fix)
  • Reactive debugging documentation
  • No clear approval gates
